Unit Testing

Einführng

Ziele

  • Code-Qualität verbessern

TDD

Test Driven Development

  • Zuerst Tests schreiben, dann Code
  • Alle Tests müssen am Anfang fehlschlagen
  • Dann wird Code geschrieben, bis der Test erfolgreich abläuft
  • Zum Schluss wird entweder der Code verbessert, oder zur nächsten Unit weitergegangen

Einfaches Beispiel

[Test]
public void IsValidExtension() 
{
	bool valid = File.IsValidExtension("myFile.ml");
	Assert.IsTrue(valid);
}

Assert-Methoden

  • IsTrue
  • IsFalse
  • Equal
  • NotEqual

Stubs / Mocks / Fakes

Vollständig isoliertes testen is so nicht möglich! Darum brauchen wir diese 3 Konstrukte

Stubs

  • Programmcode, der anstelle eines anderen Programmcodes steht
  • Code der ersetzt wird wurde noch nicht entwickelt

Mocks

  • Platzhalter für echte Objekte

Fakes
